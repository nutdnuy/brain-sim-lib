# BRAIN Simulation Settings And Data Fields

This page explains the settings shown in the WorldQuant BRAIN simulation UI and how those settings map to `brain-sim` Excel/API fields.

Use this page with:

- `examples/Tutorial 9 - Simulation Settings Deep Dive.ipynb`
- `examples/Tutorial 10 - Data Fields And Field Types.ipynb`

This is an unofficial learning reference. Treat live platform rules as the source of truth when BRAIN changes.

## Quick Mental Model

An Alpha expression is evaluated into a daily vector across a selected universe. BRAIN then applies simulation settings such as delay, neutralization, decay, truncation, pasteurization, NaN handling, and unit handling before producing portfolio weights, PnL, metrics, checks, and artifacts.

Changing settings changes the experiment. Do not compare two Alpha results unless the settings are comparable enough for the comparison to mean something.

## UI To brain-sim Mapping

| BRAIN UI label | brain-sim Excel/API field | Default in `SimulationSettings` | Type | Beginner meaning |
|---|---|---|---|---|
| Language | `language` | `FASTEXPR` | text | Expression language. Current tutorials use Fast Expression. |
| Instrument Type | `instrumentType` | `EQUITY` | text | Asset class. Current package defaults to equity simulation. |
| Region | `region` | `USA` | text | Country or regional market coverage. |
| Delay | `delay` | `1` | integer | Data timing. Delay 1 uses prior-day information for next-day trading. |
| Universe | `universe` | `TOP3000` | text | Stock set used in the simulation. Affects liquidity, coverage, and sub-universe checks. |
| Neutralization | `neutralization` | `SUBINDUSTRY` | text | Removes group-level exposure from the Alpha vector. |
| Decay | `decay` | `15` | integer | Smooths the Alpha vector through time. Can reduce turnover but can weaken fast signals. |
| Truncation | `truncation` | `0.08` | float | Caps extreme single-name weights before final portfolio construction. |
| Pasteurization | `pasteurization` | `ON` | text | Controls whether data outside the selected universe is treated as unavailable. |
| Unit Handling | `unitHandling` | `VERIFY` | text | Warns about incompatible units in expressions. |
| NaN Handling | `nanHandling` | `OFF` | text | Controls whether missing values are preserved or handled by platform rules. |
| Test Period | `testPeriod` | `P1Y6M` | text | Train/test split display inside in-sample review. |
| Max Trade | `maxTrade` | `ON` | text | Platform control for limiting trade size. The package passes it through as a setting. |
| Max Position | not implemented in `SimulationSettings` yet | not available | UI setting | Shown in the BRAIN UI screenshot. Add API support only after confirming exact API field name from live payloads. |
| Visualization | `visualization` | `False` | boolean | Controls whether visualization artifacts are requested. |

## Settings Explained

### `language`

Use `FASTEXPR` for standard WorldQuant BRAIN Fast Expression formulas. Do not mix Python syntax with Fast Expression syntax.

### `instrumentType`

The current library defaults to `EQUITY`. Keep this fixed unless the API documentation and your account explicitly support another instrument type.

### `region`

Region defines the market. It changes field availability, universe names, liquidity, calendar behavior, and evaluation context.

Beginner default: `USA`.

### `universe`

Universe defines which instruments are eligible. Larger universes improve breadth but can include less liquid names. Smaller universes are often more liquid but can make signals more crowded or harder to diversify.

Beginner default: `TOP3000`.

### `delay`

Delay controls when information is assumed to be tradable.

- `delay=1`: safer default for prior-day information.
- `delay=0`: same-day style work with stricter standards and higher risk of timing mistakes.

Beginner rule: use Delay 1 until you can explain why same-day timing is valid.

### `neutralization`

Neutralization removes average exposure inside a group. It is part of the Alpha definition, not a cosmetic setting.

Common levels:

- `MARKET`: removes broad market direction.
- `SECTOR`: keeps sector-relative effects.
- `INDUSTRY`: stronger peer comparison.
- `SUBINDUSTRY`: most granular common default.

Changing neutralization can make the same expression behave like a different Alpha.

### `decay`

Decay smooths recent Alpha vectors. It can reduce turnover because target weights change more gradually.

Beginner rule:

- Fast price/event signal: start with low decay.
- Slow fundamental signal: moderate decay can make sense.
- If turnover is high, try decay changes one at a time.

### `truncation`

Truncation limits extreme single-name weight contribution. It can reduce concentration risk, but it does not fix a weak or overly narrow Alpha thesis.

### `pasteurization`

Pasteurization affects whether data outside the selected universe can influence results. Treat it as part of the experiment because it changes coverage and peer comparison mechanics.

### `unitHandling`

Unit handling checks whether expression components combine compatible units. Warnings should be treated as signal-quality feedback.

Beginner rule: prefer ranking or z-scoring before combining fields with different units.

### `nanHandling`

NaN handling affects missing data. Turning missing handling on can improve apparent coverage, but it can also blur the difference between true zero-like values and unavailable data.

Beginner rule: missingness can itself be information. Do not hide it accidentally.

### `testPeriod`

Test period is a visible split inside the platform's in-sample analysis. It is useful for overfit awareness but does not replace hidden out-of-sample checks.

### `maxTrade`

Max Trade is passed through as `maxTrade`. It is a portfolio-construction constraint. Keep the package default unless you have a specific reason to test trade-size constraints.

### Max Position

The screenshot shows Max Position, but the current `brain-sim` model does not include a `maxPosition` field. Do not invent the API field name. Add it only after capturing or confirming the exact BRAIN API payload field.

### `visualization`

Visualization is off by default in the library. Turn it on only when you need visual artifacts and understand the API behavior.

## Data Field Types

### Matrix data field

A Matrix data field has one value per instrument-date. These fields can usually be used with standard cross-sectional operators such as `rank`, `zscore`, and time-series operators such as `ts_mean`.

Beginner checks:

- coverage
- cadence
- unit
- region
- delay availability
- universe availability

### Vector data field

A Vector data field has multiple values per instrument-date. Reduce it before using normal matrix-style operators.

Common vector reducers:

- `vec_avg`
- `vec_sum`
- `vec_count`
- `vec_min`
- `vec_max`
- `vec_stddev`
- `vec_range`

Bad pattern:

```text
rank(vector_field)
```

Better pattern:

```text
rank(vec_avg(vector_field))
```

### Group data field

A Group data field labels instruments into groups. Use it with group operators such as:

- `group_rank`
- `group_zscore`
- `group_neutralize`
- `group_backfill`

Sparse groups can break intuition. Use `densify` or broader groups only when the data mechanics justify it.

## Data Explorer Checklist

Before writing an expression, inspect:

1. Field name and description.
2. Field type: Matrix, Vector, or Group.
3. Region availability.
4. Delay availability.
5. Universe coverage.
6. Update cadence: daily, event-driven, quarterly, estimate revision, or sparse.
7. Unit and scale.
8. Missing-value pattern.
9. Dataset family.
10. Whether a vector reducer, group operator, or backfill is needed.

## Beginner Defaults

For a first live run:

| Setting | Suggested starting value |
|---|---|
| `language` | `FASTEXPR` |
| `instrumentType` | `EQUITY` |
| `region` | `USA` |
| `universe` | `TOP3000` |
| `delay` | `1` |
| `decay` | `5` to `15` |
| `neutralization` | `INDUSTRY` or `SUBINDUSTRY` |
| `truncation` | `0.06` to `0.08` |
| `pasteurization` | `ON` |
| `unitHandling` | `VERIFY` |
| `nanHandling` | `OFF` |
| `maxTrade` | `ON` |
| `visualization` | `False` |

## Common Mistakes

| Mistake | Why it hurts | Better action |
|---|---|---|
| Changing expression and settings at the same time | You cannot tell what improved the Alpha. | Change one axis per experiment. |
| Using vector fields without reduction | Operators may be invalid or meaningless. | Use `vec_avg`, `vec_count`, or another vector operator first. |
| Ignoring field cadence | Quarterly fields and daily price fields behave differently. | Match windows and decay to update frequency. |
| Treating neutralization as cosmetic | It changes portfolio exposure and PnL source. | Record neutralization as part of Alpha identity. |
| Turning NaN handling on blindly | Missingness may be the real signal. | Inspect missing pattern first. |
| Comparing different universes directly | Liquidity and coverage changed. | Compare within matched settings first. |
| Overusing decay | Turnover may fall while signal disappears. | Verify Sharpe, Fitness, Turnover, and PnL shape together. |
